📐 Structure des 9 nouvelles questions par page

Distribution des questions (Q7-Q15) :

  • Q7-Q10 (4 questions) : Concepts avancés
    • Q7 = Performance/timing/SLA concept
    • Q8 = Cas pratique / scenario réaliste
    • Q9 = Piège courant / common mistake
    • Q10 = Troubleshooting / diagnostic
  • Q11-Q13 (3 questions) : Anti-patterns & Best practices
    • Q11 = Anti-pattern (ce qui casse)
    • Q12 = Best practice (ce qui marche)
    • Q13 = KPI / Examen certification
  • Q14-Q15 (2 questions) : Décisions architecturales
    • Q14 = Scenario réaliste (production issue)
    • Q15 = Choix architectural (outil, stratégie)

🔧 Template HTML (copier-coller)

Chaque question suit ce format (identique pour Q7, Q8, ... Q15) :
<div class="qcm-question" data-page="p1"> <p class="q-text">7) Question ici...</p> <div class="qcm-options"> <div class="qcm-option"><input type="radio" name="q1-7" value="a" id="q1-7a"> <label for="q1-7a">Option A</label></div> <div class="qcm-option"><input type="radio" name="q1-7" value="b" id="q1-7b"> <label for="q1-7b">Option B</label></div> <div class="qcm-option"><input type="radio" name="q1-7" value="c" id="q1-7c"> <label for="q1-7c">Option C (BONNE REPONSE)</label></div> <div class="qcm-option"><input type="radio" name="q1-7" value="d" id="q1-7d"> <label for="q1-7d">Option D</label></div> </div> <button class="btn-validate" onclick="checkAnswer('q1-7','c',this,'p1')"> Valider </button> <div class="feedback"></div> </div>
Changements clés par question :
  • Q7 : name=q1-7, id=q1-7a/b/c/d, onclick checkAnswer('q1-7','c'...)
  • Q8 : name=q1-8, id=q1-8a/b/c/d, onclick checkAnswer('q1-8','b'...)
  • ... continuer jusqu'à Q15

💬 Ajouter justifications (JavaScript)

À la fin du fichier, dans le bloc const justifications = { }, ajouter :
const justifications = { // Existantes (copier depuis fichier) 'q1-1': "CI signifie...", ... 'q1-6': "...", // NOUVELLES 'q1-7': "Justification pour Q7 (2-3 phrases)", 'q1-8': "Justification pour Q8 (2-3 phrases)", 'q1-9': "Justification pour Q9 (2-3 phrases)", ... 'q1-15': "Justification pour Q15 (2-3 phrases)" };

📖 Enrichir les Concepts (+2-3 paragraphes)

Pour chaque page, ajouter APRÈS les paragraphes existants :
<p><strong>Titre 1 (concept avancé) :</strong> Texte détaillant un concept pas couvert précédemment... (3-5 phrases)</p> <p><strong>Titre 2 (anti-patterns) :</strong> Expliquer les 3 erreurs courantes et leurs risques... (3-5 phrases)</p> <p><strong>Titre 3 (ROI ou KPI) :</strong> Mesures clés, impact business... (3-5 phrases)</p>

✅ Checklist qualité (par question)

Avant de valider chaque question, vérifier :
  • ✓ 4 options A/B/C/D présentes
  • ✓ UNE SEULE bonne réponse
  • ✓ Options plausibles (piègent si mauvaise compréhension)
  • ✓ Justification pertinente (2-3 phrases)
  • ✓ HTML syntaxe valide (pas d'erreurs)
  • ✓ ID cohérent : name="q{page}-{num}" et onclick checkAnswer('q{page}-{num}')
  • ✓ Question teste concept, pas juste un "fait"
  • ✓ Réponse n'est pas évidente (force réflexion)
